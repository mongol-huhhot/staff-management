# 4. 開発手順

## 4.1 新規アプリ作成

例：資格管理アプリ（qualification）

### Step1 アプリ登録

アプリケーションを登録して管理するテーブルはmaster_schema.mtb_item_categoryです。

master_schema.mtb_item_category

```sql
INSERT INTO master_schema.mtb_item_category
(
    category_code, -- アプリ記号
    category_name, -- アプリ名
    sub_category_code, -- アプリの機能の記号。複数機能がある場合、機能別にTAB分けて表示
    sub_category_name, -- アプリの機能の名。複数機能がある場合、機能別にTAB分けて表示するタブ名
    remarks,           --  備考
    data_structure,    -- single(シングルフォーム)/repeatable(複数フォーム(例：一人複数銀行あり))/page(自動フォーム生成ではなく、手動作成フォームを適用)
    ui_component,      -- data_structure=pageの場合手動作成コンポネントの登録名(main.jsに追加：
                       --     import ImageUploader from '@/components/forms/fields/ImageUploader.vue' ① 
                       --     const vuetify = createVuetify({
                       -- components: { ...components, 
                       --     VDateInput,  ScopeRuleEditor, AppProcessSelector, AppRolePermissionPage, ImageUploader, },　②
                       -- directives,
                       -- locale: {
                       --     locale: 'ja',
                       --     fallback: 'en',
                       --     messages: {
                       --     ja: customJa,
                       --     },
                       -- },
                       -- })
            　　　　　 --　 　　　)

    enabled,           -- 有効(active)/無効(inactive)
    show_order         -- 表示順番（複数ある場合TABの順番）
)
VALUES
(
    'qualification',　    -- アプリ記号
    '資格管理',         　-- アプリ名
    'qualification',      -- 機能記号
    '資格管理',           -- 機能名
    'repeatable',         -- 重複可  
    null,                 -- 自動生成のため手動コンポネントなし
    'active',             -- 有効
    100                   -- 表示順（一つの場合、数値と関係はない第一TABに表示、複数ある場合小さいから大きい順位にTABに表示）
);
```

---

### Step2 メニュー登録

普段public/js/index.jsにMAIN_CONFIG.tab2sqltag_listへ追加

```javascript
qualification: {
    label: '資格管理',
    data_key: 'qualification',
    jsonb_fields: ['profile_jsonb'],// jsonb カラムの一覧
    separate_items: ['staff_code', 'profile_version'],// jsonb以外の普通カラム    
    skip_reload: true, // 毎回リロードしない
    sqltags: {  // CRUD　
        select: 'qualification.get',
        save: 'qualification.save', // upsert
        delete: 'qualification.delete'
    }
}

```

----------------------------------------------------------------------------


### Step3 SQLTAG作成

sql テンプレートファイルにSQLTAG追加

```text
qualification.get
qualification.save
qualification.delete
```

---

### Step4 項目定義登録

mtb_item_dictionaryへ登録  

フォームに表示するため、またが仕様として必ずこの辞書に項目を登録する  
仮に手動作成コンポネントでもフォームの項目は登録する   

- １）項目の閲覧・編集権限管理に使われる
- ２）自動フォーム生成に使われる
- ３）システムの技術仕様として管理

画面が自動生成されることを確認する。

Vueソース追加不要。


### 辞書アプリ構造

```text
辞書管理アプリ
 ├─ アプリ/機能一覧
 │   └─ mtb_item_category
 │
 ├─ 項目一覧
 │   └─ mtb_item_dictionary
 │
 ├─ 項目編集フォーム
 │   ├─ 基本情報
 │   ├─ 表示・入力定義 field_definition
 │   ├─ 選択肢・説明 item_description
 │   └─ 計算式 formula
 │
 └─ プレビュー
     └─ 実際の DynamicVuetifyForm で表示確認

```
