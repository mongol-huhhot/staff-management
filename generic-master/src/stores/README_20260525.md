# SURUPAs databaseEngine 整理版

含まれるもの:

- `php/databaseEngine.php` 新入口
- `php/requestHandler.php` 既存 URL 互換入口
- `php/SqlRequestHandler.php` multi SQL / LOOP execute / 条件タグ対応
- `php/auth.php` JWT / public SQL tag / development AUTH_SKIP 対応
- `php/login.php`, `verify.php`, `logout.php`
- `js/useDbStore.js` Pinia 統合 Store
- `js/BaseStore.js` 既存 useBaseStore 互換 wrapper
- `js/main.webcomponent.js` WebComponent 用 entry
- `js/vite.config.js` Vite 移行例

重要:

1. 既存の multi SQL は `dbAccessWithMultiTags()` / `executeMultiQuery()` で維持。
2. 既存の LOOP execute は `loopExecute()` または各 SQL tag に `LOOP: [...]` を渡す形で維持。
3. ログイン・登録・メール認証など public SQL tag は token なしで通す。
4. 本番で `AUTH_SKIP=true` は無効。開発環境のみ有効。
5. `requestHandler.php` は互換用に残すため、既存フロントの URL はすぐ壊れない。
