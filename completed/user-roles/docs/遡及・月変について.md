## 遡及入力内容  
遡及にあたり、差分データの提供は２つある

### 遡及データ概要  
```text
③ 変動・遡及の対応方法（整理版）

変動CSV取込みデータは salary_schema.tbxx_monthly_revisions.csv_uploaded に保存する。

CSV取込みデータは (staff_code, yearmonth, item_code) 単位で集計し、その差分値をcsv_agg_valuesカラムに保存する。

遡及手入力データは input_values に保存する。手入力対象項目から算出した報酬対象額差分（ΔJ9107）も input_values 内に保存する。

再計算対象は salary_schema.tbxx_monthly_revisions に存在する (staff_code, yearmonth) のみとする。

再計算時のベースデータは、通常給与計算結果テーブル salary_schema.tbxx_payroll_calculation_result の該当月データとする。

CSV差分は、各給与計算項目に対する差分値として再計算に反映する。

手入力差分は、算出済みの報酬対象額差分（ΔJ9107）として報酬対象額（J9107）に加算する。

再計算結果は salary_schema.tbxx_payroll_retroactive_result に保存する。

遡及入力のみ、CSV取込みのみ、両方あり、のいずれの場合でも、再計算処理の流れは共通とする。
```  


細かい内容は次の説明を確認して下さい  

- CSV取り込みデータはテーブルsalary_schema.tbxx_monthly_revisionsのcsv_uploadedカラムに保存
- CSV取り込みデータを(staff_code, yearmonth, item_code)別に合計した差分額（Δitem_code）はcal_valuesカラムに保存
- 遡及入力データはinput_valuesカラムの保存。合計計算して入力報酬対象額差分(ΔJ9107)もカラムinput_valuesに項目J9107として保存。理由は１つ報酬対象額差分(ΔJ9107)しかないからである

### ② 手入力
現在の手入力は下記の項目に限定する。これらの値をまとめて合計計算して報酬対象額差分(ΔJ9107)を求める  

- 遡及入力内容は差分として入力される。  
  - 項目：
    - A. 食事代(JC180) = 食事単価(J8021) * 食事日数（回数）(JE220)
    - B. 社宅現物(JS002)
    - C. 報酬現金（報酬対象額）(J9107)
    - D. 月額変動金額（J9107）
  - 上記のを合計（A+B+C+D）して（報酬対象額差分）（ΔJ9107）（報酬対象額（J9107）差分）


## ③ 変動・遡及の対応方法  
- CSV項目について、(staff_code, yearmonth, item_code)の差分額（Δitem_code）を給与計算項目に足して、マスタ・雇用計算・勤怠データは通常給与計算のままで、給与を再計算する。結果はテーブルsalary_schema.tbxx_payroll_retroactive_resultのcal_valueとsalary_resultにそれぞれ保存
- 入力値について、入力しながら、報酬対象額（J9107）差分ΔJ9107を報酬対象額（J9107）に足す。
- つまり、CSV取込みデータがある場合、再計算後の報酬対象額（J9107）に入力結果の報酬対象額差分ΔJ9107を出す。他は変わらない。ということは、現在の給与計算（遡及がある場合遡及の遡及がない場合通常給与計算の）の報酬対象額（J9107）に入力値の差分ΔJ9107を足すということです。
- 遡及入力、変動CSV取込みに関わる計算はテーブルsalary_schema.tbxx_payroll_retroactive_resultになります。遡及入力、あるいは変動CSV取込みのかた方があっても計算方法には変化がない

## 短時間労働者  

① 短時間労働者はおります。
　雇用契約書より算出した、月の所定労働時間が130Ｈ以下の者が対象です。
　（月の所定労働時間→年間所定労働日数÷52週×1日の実働時間×4週）

### 計算方法  

PG SQLで雇用契約テーブルからデータ取得方法  

```python
WITH w_src AS (
    SELECT DISTINCT ON (vlc.staff_id)
        vlc.*,
        COALESCE(NULLIF(vlc.contract_json->>'year_work_day', ''), '0')::numeric AS JE020,   -- JE020、年間所定労働日数
        COALESCE(NULLIF(vlc.contract_json->>'total_work_time', ''), '0')::numeric AS JE080  -- JE080、実労働時間
    FROM public.tb_contract AS vlc
    ORDER BY
        vlc.staff_id,
        vlc.contract_date DESC,
        vlc.create_date DESC NULLS LAST
)
select
    (JE020 / 52.0) * JE080 * 4.0 as monthly_work_hours, -- 月間労働時間(年勤務日数/年52週(=週平均日数)*実労働時間（=週労働時間）*4.0(月４週、=月労働時間))
    CASE
        WHEN (JE020 / 52.0) * JE080 * 4.0 <= 130.0 THEN '1'
        ELSE '0'
    END AS "JI006",　-- JE020 / 52.0　週間平均労働日数（年52週）、実労働時間
    w_src.*
FROM w_src;

```

|1|salary_schema.tbxx_monthly_revisions|月間変動データCSV取込み・手入力のオリジナルデータを保存するテーブル。”変更データ”取込みに由来するが、今月のデータは給与計算に適用され、”過去”のデータのみがこのテーブルのcsv_uploadに入る|
|2|salary_schema.tbxx_monthly_change_calc_data|給与・import/inputデータから月変の計算でターゲットを絞る。計算して月変対象になりそうなスタッフ一覧。全然対象になりそうもないスタッフのデータはこのテーブルに保存しない。|

with w_staff as(

    CASE
        WHEN birthday >= DATE '2019-05-01' THEN '5'
        WHEN birthday >= DATE '1989-01-08' THEN '4'
