# マスタ設計
## 方針
- マスタデータを適用日でトランザクションデータとして管理する
- 編集データ中データを本番データを別テーブルに管理する
- 承認、あるいは、確定した場合データを本番テーブルに反映する

## 設計概要
- 個人情報ステータス： draft, submitted, approved, rejected, cancelled
- 承認は一次のみ
- 承認ある場合、approved時に編集データを本番データにコピー
- 承認なしの場合はsubmitted時に編集データを本番データにコピー
- rejected, cancelledは管理者の操作でステータスがdraft, submittedの場合のみできます
- cancelled時、再提出と

### 月内で有効だったスタッフ情報取得  

```sql
WITH params AS (
  SELECT
    DATE '2026-02-01' AS month_start,
    (DATE '2026-02-01' + INTERVAL '1 month - 1 day')::date AS month_end
),
hist AS (
  SELECT
    d.*,
    LEAD(d.valid_from) OVER (
      PARTITION BY d.staff_id
      ORDER BY d.valid_from
    ) - 1 AS valid_to
  FROM staff_schema.staff_profile_data d
)
SELECT h.*
FROM hist h
CROSS JOIN params p
WHERE h.valid_from <= p.month_end
  AND COALESCE(h.valid_to, DATE '9999-12-31') >= p.month_start
ORDER BY h.staff_code, h.valid_from

```
